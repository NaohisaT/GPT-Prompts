# **CR3 v1.1：レセスタ対応コメント生成ルール（改訂版）**

---

## **目的**

レセスタによる自動抽出結果に対し、医療機関向けの\*\*CRコメント（Consulting Report コメント）\*\*を作成する。  
 本ガイドラインは、診療報酬制度に基づく算定支援・整合性確認コメントを一貫した品質で出力するための標準仕様を定める。

---

## **入力仕様**

CRコメント生成には、以下の入力情報を前提とする。

| 項目 | 内容 | 必須 |
| ----- | ----- | ----- |
| ①分類 | 「整合性」または「算定支援」 | ✅ |
| ②レセスタメッセージ | レセスタ抽出の制度趣旨・説明 | ✅ |
| ③施設基準・算定基準 | 厚労省告示・通知・事務連絡など制度的根拠 | ✅ |
| ④レセスタ抽出条件 | 対象患者の抽出ロジック（and/or構文等） | 任意 |
| ⑤該当レセプト内容 | 算定状況確認・比較情報 | 任意 |

---

## **💬 出力仕様（コメント構造）**

出力コメントは **必ず「●」で始まる3文構成** とし、  
 各文はおおむね **80字以内** に収めること。

### **🧩 基本構成ルール**

| 行 | 内容 | 記述目的 |
| ----- | ----- | ----- |
| 1文目 | 制度趣旨・点数概要 | 制度的根拠の整理 |
| 2文目 | 抽出条件・確認対象の要約 | 抽出ロジックの明示 |
| 3文目 | 判断・助言・確認の指針 | 現場対応の方向性提示 |

---

## **🧭 コメント区分ごとの作成ルール**

### **🟥 整合性コメント（査定リスク警告型）**

**目的：** 算定済項目が制度要件を満たしていない可能性を警告する。

**文体トーン：** 中立的・確認促し型。断定禁止。

**構成例：**

```
●〇〇は、△△を行った場合に算定可能な点数です。
●本件では、□□の算定がなく、要件を満たしていない可能性があります。
●査定リスクを避けるため、算定内容および算定根拠をご確認ください。
```

**使用語例：**

* 「算定できない可能性があります」

* 「要件を満たしているかご確認ください」

* 「査定状況に応じてご判断ください」

---

### **🟩 算定支援コメント（前向き提案型）**

**目的：** 要件を満たしているのに算定が漏れている可能性を指摘。

**文体トーン：** 積極的・支援的。

**構成例：**

```
●〇〇は、△△を実施した場合に算定可能な項目です。
●本件では、□□を実施しており、算定可能性があります。
●届出状況および算定要件を満たす場合は、算定をご検討ください。
```

**使用語例：**

* 「算定できる可能性があります」

* 「算定をご検討ください」

* 「届出や研修要件の確認をお願いします」

---

## **🧱 出力テンプレート（v1.1形式）**

```
#分類：整合性（または算定支援）
#レセスタメッセージ：
（レセスタ上のメッセージ内容）

#CRコメント：
●（1文目：制度趣旨・概要）
●（2文目：抽出条件・対象要約）
●（3文目：確認・助言・方向性）

#根拠：
（厚労省告示・通知番号・該当箇所）
```

---

## **📘 抽出条件の自然文化ルール**

レセスタの `含む / 含まない / and / or` の論理式を、自然文に変換する統一ルールを設ける。

| 条件式 | 自然文変換例 |
| ----- | ----- |
| A 含む and B 含まない | 「Aがあり、Bがない患者を対象としています。」 |
| A 含む or B 含む | 「AまたはBの病名・算定がある患者を抽出しています。」 |
| A 含まない | 「Aを算定していないケースを対象としています。」 |
| 同日／同月 | 「同日（または同月）に実施されているものを抽出しています。」 |

🔹 補足：抽出条件の目的は「この患者がコメント対象になった理由を説明すること」であり、技術的構文ではなく**臨床意図の日本語化**を優先する。

---

## **📜 根拠明示のルール**

コメント末尾に、厚労省通知・事務連絡・告示等の出典を**1行で明記**する。

```
（根拠：令6保医発0305･4／告示４第３・４の９⑴）
```

可能な範囲で該当ページや注番号（例：「注3」「⑵」）も添記。

---

## **🧠 トーン＆文体統一指針**

| 要素 | 整合性 | 算定支援 |
| ----- | ----- | ----- |
| トーン | 冷静・確認型 | 積極・支援型 |
| 動詞表現 | 「確認」「判断」「該当」 | 「検討」「算定」「届出」 |
| 禁止表現 | 「算定できない」「誤り」 | 「必ず算定」「確実に」 |
| 末尾統一 | 「ご確認ください」 | 「算定をご検討ください」 |

---

## **🔗 応用例：横断コメント（複合検査・同一患者群）**

同一患者・同一日で複数の整合性コメントが発生する場合は、  
 「群コメント」として統合する形式も用意。

**例：糖代謝関連検査群**

```
●グルコース・インスリン（IRI）・CPRは、耐糖能精密検査の包括対象です。
●同日に複数算定している場合は、併算定不可となる可能性があります。
●包括対象の確認および実施意図の記録状況をご確認ください。
（根拠：告示４第３・４の８⑴）
```

---

## **🪶 対外説明文テンプレート（質問対応用）**

```
【ご質問内容】
〇〇について、同様の患者が抽出されていない理由を知りたい。

【回答】
レセスタでは、△△の条件を満たし、かつ□□が算定されていない患者を抽出しています。
同様の病名を有していても、××や◇◇の要件を満たさない場合は対象外となります。
したがって、今回抽出された方は算定要件を満たしており、算定支援として表示されています。
```

このテンプレートは、医療機関からの「なぜ抽出されないのか」質問に対する標準回答文として使用。

## **🏁 総評**

本v1.1では、\*\*「現場で即使える簡潔さ」＋「監査対応に耐える制度根拠」\*\*を両立。  
 レセスタ出力の整合性コメント／算定支援コメント／CRコメント／質問対応文を  
 **一つの体系で統一**できる構造になっている。

このマニュアルに基づけば、コメント出力を完全に自動化しても、  
 人間監修レベルの正確性と自然さを両立可能。
